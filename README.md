# Jackson Ascent Lead Intelligence

Lead intelligence and prospect qualification for home-service businesses (roofing first).
Self-hosted **Next.js + TypeScript + Supabase + Apify**, deployable on Vercel. No Base44.

## What it does (V1)

- Scrapes local businesses from Google Maps via Apify
- Stores them in Supabase Postgres (`prospects`, `scrape_runs`)
- Deduplicates by Google `place_id` and **upserts** on re-scrape (refreshes data without wiping CRM fields)
- Simple scraper UI with table + CSV export

Scoring, dashboard, outreach, and pipeline come in later phases.

## Setup

### 1. Supabase
1. Create a project at https://supabase.com
2. Open **SQL Editor** and run `supabase/migrations/0001_init.sql`
3. From **Project Settings -> API**, copy the **Project URL** and the **service_role** key

### 2. Environment variables
Copy `.env.example` to `.env.local` and fill in:

```
APIFY_API_TOKEN=...
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=...
```

### 3. Run locally
```
npm install
npm run dev
```
Open http://localhost:3000

## Deploy to Vercel
1. Import this repo at https://vercel.com/new
2. Add the three env vars above in **Settings -> Environment Variables**
3. Deploy

> The service_role key bypasses row-level security and must stay server-side only.
> All Apify + Supabase access happens in API routes, never in the browser. Because
> there is no auth yet (V1 is single-operator), keep the deployment private or add
> Supabase Auth before sharing the URL.

## Architecture

```
app/
  page.tsx                  Scraper UI (client)
  layout.tsx
  api/
    scrape/start/route.ts   Launch Apify run, create scrape_run
    scrape/check/route.ts   Poll Apify, import + upsert prospects
    prospects/route.ts      List prospects
lib/
  apify.ts                  Apify REST helpers + item mapping
  supabase.ts               Server Supabase client
supabase/migrations/        SQL schema
```
